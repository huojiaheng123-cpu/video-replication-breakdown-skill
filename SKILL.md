# 视频复刻拆解

## 目标

把一个值得模仿的视频变成可复刻的创作蓝图：证据、分镜、节奏、画面、声音、字幕、脚本、提示词和本地资产路径。

默认使用中文输出。这个 skill 专门服务“我也想做出类似视频”，不是“帮我看懂它讲了什么”。

## 边界

- 用户问“讲了什么、靠不靠谱、对我有什么启发、我该怎么做”时，改用 `watch-video`。
- 用户问“怎么拍、怎么复刻、怎么拆、怎么模仿、这个视频结构是什么、帮我生成脚本/提示词”时，使用本 skill。
- 复刻不等于抄袭：保留结构、节奏、镜头功能和创作方法，替换主题、素材、文案和表达。
- 不要只凭标题、封面或单张截图生成复刻方案；证据不足时先说明边界。
- 默认把材料放进 `00_工作台`，不要自动写入 `01_成长笔记`。

## 默认流程

1. **读取证据**
   - 优先复用 `watch-video` 的证据底座：真实浏览器打开链接、保存页面信息、关键帧、图文帧、媒体探测、音频和转写。
   - 本地视频用 FFprobe/FFmpeg 读取时长、音轨、分辨率，抽 contact sheet 和关键帧。
   - 抖音/TikTok 图文作品要提取全部图片帧、作者、发布时间、音乐、互动数据和结构化元数据。
   - 网页阻止播放、没有媒体流、只有背景音乐或只有图文时，明确说明。

2. **判断复刻价值**
   - 先判断这个视频为什么值得复刻：钩子、情绪、信息密度、镜头设计、叙事闭环、可迁移模板。
   - 如果内容只有观点价值而没有明显创作结构，建议改用 `watch-video` 做学习总结。

3. **拆创作结构**
   - 开头钩子：前 1-5 秒如何抓注意力。
   - 叙事结构：起因、推进、转折、证明、收束。
   - 分镜时间线：每段时间、画面、口播/字幕、声音、镜头功能、可复用点。
   - 风格层：画面、色彩、构图、景别、运动、剪辑节奏、字幕、人设和情绪。
   - 声音层：口播、环境声、音乐、音效、停顿、字幕和 ASR 误差。
   - 素材层：需要哪些素材、哪些片段必须保留、哪些可以替换。

4. **生成复刻资产**
   - HTML 案例拆解页：承载原视频/图文证据、分镜、镜头功能表、ASR/字幕复核、风格层、素材映射和本地路径。
   - 复刻脚本：按用户的主题生成同结构脚本，不照搬原文。
   - AI 提示词：给视频生成、剪辑、图片生成或口播改写工具使用。
   - Markdown 摘要：只作为索引和结论，不替代 HTML 证据页。

5. **交付**
   - 对话里先说明：是否真实读取、复刻价值、核心结构、下一步怎么拍。
   - 链接到 HTML 报告、证据文件、转写、联系图和复刻脚本。
   - 说明哪些部分是根据证据确认的，哪些是创作推断。

## HTML 报告结构

使用 `assets/html-report-template.html` 作为起点。报告至少包含：

- 原视频/图文证据区
- 一句话复刻价值
- 分镜时间线
- 镜头功能表
- 风格层分析
- ASR/字幕复核
- 复刻脚本
- AI 生成提示词
- 本地资产路径

详细栏目说明见 `references/report-structure.md`。

## 推荐输出结构

- 读取状态：已播放 / 已提取图文帧 / 只看到页面 / 被拦截。
- 复刻价值：这个视频为什么值得学。
- 核心结构：它的创作骨架是什么。
- 分镜摘要：按时间线列出关键镜头和功能。
- 复刻方案：换成用户主题后怎么拍、怎么说、怎么剪。
- 交付文件：HTML 报告、证据材料、脚本、提示词。
- 不确定性：没有确认的媒体、音频、字幕或页面限制。

## 常见错误

- 把学习总结当复刻拆解：如果没有镜头、节奏、脚本和提示词，就不是本 skill 的完整输出。
- 把复刻做成照抄：保留结构，替换主题和表达。
- 只分析口播，忽略画面和声音：复刻视频必须看画面、字幕、环境声和剪辑。
- 用 Markdown 堆长文代替 HTML：复刻拆解需要证据可视化，复杂案例默认生成 HTML。
