# HTML 案例拆解页结构

## 必填栏目

- 基本信息：标题、作者、平台、链接、发布时间、时长或图文帧数、读取状态。
- 证据区：原视频、图文帧、contact sheet、关键帧、转写和页面元数据路径。
- 复刻价值：一句话说明这个视频可迁移的创作方法。
- 分镜时间线：时间段、画面、口播/字幕、声音、镜头功能、可复用点。
- 镜头功能表：钩子、铺垫、证明、转折、情绪推进、行动引导、收尾。
- 风格层：画面风格、色彩光线、构图景别、镜头运动、剪辑节奏、字幕风格、声音设计。
- ASR/字幕复核：原始 ASR、硬字幕、人工修正和不确定句。
- 复刻脚本：按用户主题改写后的开头、主体、转折和结尾。
- AI 提示词：视频生成、图像生成、口播改写或剪辑指令。
- 本地资产路径：素材包、分析包、分镜片段、转写、SRT、HTML 报告。

## 图文作品处理

抖音图文、TikTok carousel 或网页相册没有连续视频时：

- 把每张图当作一帧或一屏报告证据。
- 提取图片顺序、分辨率、作者、音乐、互动数据和页面风险提示。
- 如果背景音乐可读取，只描述音乐信息，不把音乐当作口播。
- 输出“图文帧时间线”，不要虚构视频时长。

## 复刻脚本要求

- 保留原视频的结构功能，不照搬原句。
- 明确哪些镜头必须拍、哪些可以用图片/素材替代。
- 给出 15-30 秒、45-60 秒或原长度版本，按用户需求选择。
- 每段写清楚画面、口播/字幕、声音和剪辑。

## 质量门槛

- 不能只有观点总结。
- 必须有证据路径或明确的不确定性说明。
- 必须输出至少一个可执行复刻脚本或提示词。
- 复杂案例必须生成 HTML，简单案例可以先给 Markdown 版蓝图。
